## 第1章に向けて: 導入と全体像

### Q1. マルチコプタはなぜ「手放し」では飛べないのか？

（学生の疑問: 紙飛行機やグライダーは制御なしでも飛ぶ。気球も浮くだけなら制御は要らない。
マルチコプタは何が違って、コンピュータによる制御が「絶対に」必要なのか？）

**先生の回答:**（2026-07-13）

- 全てのプロペラが全く同じもので、同じモータが回しているなら、ひっくり返らずに飛べるかもしれない。
  しかし実際はすべて微妙に違っていて、どうしてもわずかな**推力差**が発生する。
  それによって、わずかな**姿勢を崩すトルク差**が発生する。
- わずかな違いでも、空中に浮かぶマルチコプタにとっては致命的。
  姿勢はどんどん崩れ、いずれ**ひっくり返ってしまう**。
- したがって、姿勢の崩れを**検知**して、それを**打ち消すようにプロペラの推力差を補正**してやる必要がある。
- 一方、通常の飛行機は、姿勢を崩すトルクが生じて姿勢が崩れ始めても、
  それを復元させるトルクを**機体の構造が**発生させることができる。
  - そういう復元させるトルクを**復元力**と言う。
  - 復元力が発生することを**静安定**と呼ぶ。
- 飛行機や気球には静安定があるが、マルチコプタは自らの姿勢の崩れに対して
  自らを戻す復元力を**構造上発生させることができない**。
  だから姿勢を検知してコンピュータが制御してやる必要がある。

**記事化メモ:**
- キーワード: 推力差 → トルク差 → 姿勢の崩れの発散 / 復元力 / 静安定
- 対比構造が使える: 飛行機・気球（静安定あり）vs マルチコプタ（静安定なし）
- 教材アイデア: わずかな推力アンバランスを与えた機体の 2D シミュレーション。
  「制御 OFF」だと姿勢が発散してひっくり返り、「制御 ON」にすると水平を保つトグル。
  推力アンバランス量をスライダーで変えられると効果的。

### Q1-2. 飛行機はなぜ構造だけで復元力を発生できるのか？

**先生の回答:**（2026-07-14）

- 固定翼機は**上反角**（両方の翼を上に傾ける）によって復元力を得ている。
- 仕組み: 機体がどちらかに傾くと、揚力に横方向成分が生じて機体が**横に滑り始める**。
  これによって横から風を受けることになる。
- 左右に上反角がついていると、その横から受ける風の向きに左右の翼で違いが生じる。
  翼の揚力は風の速さと向きで決定的に変わるので、横風によって**左右の翼の揚力に差**が生じ、
  それが傾きを修正する復元力となって、傾きを自動的に修正する。
- マルチコプタには揚力を稼ぐ**固定翼がない**ので、上反角効果を利用した復元力は付けられない。
- 固定翼を持つハイブリッド機も存在するが、今回の議論では除外する。

**記事化メモ:**
- 復元のメカニズムは「傾く → 横滑り → 横風 → 左右の揚力差 → 復元トルク」という因果連鎖。
  記事では矢印図か、ステップ送りのアニメーション図解が向く。
- 「静安定は翼がタダで実行してくれるフィードバック制御」という見方もできそう（先生に確認）。

### Q1-3. 姿勢の崩れはどのくらいの速さで進むのか？（StampFly で定量化）

**先生の指示:**（2026-07-14）
簡易計算に必要なのはロール軸慣性モーメント・プロペラと重心の距離・推力係数。
プロペラは角速度の2乗×推力係数に比例した推力を出す（T = Ct·ω²）。
StampFly の値を使い、左右のプロペラ角速度がわずか 0.1% ズレていた場合に
水平から 90° に傾くまでの時間を計算せよ（計算実務は学生役=AI が担当）。

**使用パラメータ（すべて先生のリポジトリの一次情報から回収）:**

| 量 | 値 | 出所 |
|----|-----|------|
| 質量 m | 36.8 g | `M5StamFly_spec_ja.md` 製品重量（firmware config.hpp は 0.037 kg でほぼ一致） |
| ロール慣性 Ixx | 9.16e-6 kg·m² | `firmware/vehicle/tasks/api_task.cpp` kSpecInertia[0]（workshop lesson_06 の表と一致） |
| モーメントアーム d | 0.023 m | `sf_algo_control/control_allocation.hpp` QuadConfig（X配置、モータ位置 ±23 mm） |
| 推力係数 Ct | 1.0e-8 N/(rad/s)² | `sf_algo_control/motor_model.cpp` |

**計算**（スクリプト: `notes/calc/roll_divergence.py`、仮定も同ファイル冒頭に明記）:

- ホバリング: 1モータあたり T₀ = mg/4 = 90.3 mN、ω₀ = √(T₀/Ct) ≈ 3004 rad/s（約 28,700 rpm）
- 左2発を ω₀(1+0.001)、右2発を ω₀ とすると、1モータあたりの推力差 ΔT ≈ 2·δ·T₀ = 0.18 mN（**18 mgf、米粒半分ほどの重さ相当**）
- ロールトルク τ = 2·ΔT·d = 8.3e-6 N·m → 角加速度 α = τ/Ixx ≈ 0.91 rad/s²
- 一定トルクなので φ(t) = ½αt²

| 傾き | 到達時間 |
|------|---------|
| 1° | 0.20 s |
| 5° | 0.44 s |
| 10° | 0.62 s |
| 45° | 1.32 s |
| **90°（ひっくり返る）** | **1.86 s** |

**考察メモ:**
- 0.1% のズレ（3004 rad/s 中のわずか 3 rad/s ≈ 29 rpm）ですら約 1.9 秒で横転。
  実機のプロペラ・モータ個体差は % オーダーのはずで（先生に要確認）、δ=1% なら時間は 1/√10 で
  **90° まで 0.59 s、10° まで 0.20 s** と人間の反応時間そのものになる。
- 人間の視覚反応（~0.2 s）で 1° の傾きに気づけたとしても、必要な補正は「スロットル差 0.1% 未満の
  精度で、連続的に、永久に」であり、プロポのスティックでは実現不可能。不安定平衡の維持は
  鉛筆を指先で立てるのと同じ構造。
- **検算として重要:** 最終結果は Ct に依存しない。ホバリング条件 Ct·ω₀² = mg/4 により
  τ = mg·δ·d となって Ct と ω₀ が打ち消し合う（Ct は ω₀ の表示にのみ使用）。
  つまり Ct の精度が結果を左右しない、頑健な見積もり。
- 未確認事項: 空力減衰・並進運動との連成・ジャイロ効果は無視した（発散初期では小さい）。
  実機の個体差 δ の実測値は未確認。

**教材アイデア（第1章の目玉候補）:**
δ をスライダーで変えると横転までの時間が変わる 2D シミュレーション。
「制御 OFF」でこの計算の通り発散し、「制御 ON」で水平を保つ対比を体感させる。

## 第1章に向けて: なぜロータは4つか

### Q2. なぜロータは4つなのか？

**先生の回答:**（2026-07-14）

- マルチロータを三次元空間で自由に飛ばすために最低限必要なのは、
  **上下の制御**と、**X軸・Y軸・Z軸周りの回転の制御**の**4つ**。
- 「前後左右の制御は？」と思うかもしれないが、**前後は機体を前に傾けると推力が前に傾いて
  前に進む**。左右も左右に傾けることで可能。
  したがって上下と三軸回りの回転の4つを制御できればよい。
- 簡単に言うと、**4つの量を制御するためには、プロペラ4つをそれぞれ独立に制御する必要がある**。
  マルチコプタは4つの量を制御することで自由に飛び回れる。
- 4つの量を自由に制御できることを**自由度が4**という。
  **自由に動かせるプロペラの数と自由度の数が最低でも揃っていないと制御はできない**。
  これがロータが4つの理由。

**記事化メモ:**
- 「操りたい量4つ（上下・ロール・ピッチ・ヨー）⇔ 独立な入力4つ」の対応表が図解に向く。
- 前後左右は「傾ける→推力ベクトルが傾く→水平成分で進む」という間接制御。
  機体の位置・姿勢は6量あるのに独立入力は4つ、という構図は後章（劣駆動）への伏線。
- 教材アイデア: 推力ベクトルの傾きで前後移動が生まれる矢印図（インタラクティブ）。

### Q2-2. トライコプタ・ヘキサ・オクタは？

**先生の回答:**（2026-07-14）

- トライコプタ: サーボでロータを傾ける機構が4つ目の入力、という理解で**良い**。
- ヘキサ・オクタの意味は二つ:
  1. **制御の冗長性**を持たせて安全にする。
  2. より本質的には、三次元空間の運動は本来
     **並進3自由度（上下・前後・左右）+ 各軸回りの回転3自由度 = 合計6自由度**。
     ヘキサ以上のマルチコプタは**ロータの配置や傾きを最適にすることで、
     この6自由度を自由に制御することが可能**になる。

**記事化メモ:** 1.6節のコラムに反映済み。「6自由度の完全制御（full actuation）」は
先生の研究テーマ（fixed-tilt-fully-actuated-multirotor）に直結する話題なので、
発展章の候補として温存。クアッドの「傾けて進む=劣駆動」との対比が軸になる。

## 第2章に向けて: 反トルクとロータ配置

### Q2-4. 反トルクとは何か？ なぜ対角配置か？

**先生の回答:**（2026-07-14）

プロペラの反トルクは実は**二つの物理現象**。

1. **空気による反作用の反トルク**
   - プロペラは回転するとき空気を押しのける。作用反作用の法則により、
     空気を押しのけるプロペラは同時に空気に押されている。
   - CW に空気を押しのけているプロペラは、空気から CCW に押される。これが反トルク。
   - CW に強く回すほど、プロペラは空気に CCW に回されようとする。
   - モータとプロペラは機体フレームに固定されているので、プロペラを CW に回すと
     **空気が機体を CCW に回そうとする**。
   - 同じ回転方向のプロペラだけを使うと機体は常にプロペラと反対方向に回ろうとする。
     だから**違う回転方向のプロペラを同じ数だけ用意して回転を打ち消す**。
2. **回転体が加速するときの加速トルク（慣性反トルク）**
   - モータなどの回転体を机の上に置いて始動すると、起動時に身震いするように
     机の上で一瞬回転する。あれが加速による反トルク。
   - **空気の反作用よりはるかに速く起きる現象**。
- **ヨーの制御は、空気と加速による二つの反トルクを上手に使うことがポイント**。

**なぜ対角配置か:**
- ロール・ピッチの制御では、右と左（または前と後）のペアを強めたり弱めたりする。
- 対角を同じ回転方向にしておくと、ロールでもピッチでも
  **CW と CCW がちょうど同じように強く/弱くなる**ので、ヨー回転のバランスは
  取れたまま。**ロールやピッチを回すときにヨーが同時に起こることがなくなる**。

**記事化メモ:**
- 「二つの反トルク」の区別は教科書にあまりない切り口で、章の目玉にできる。
  - 空気の反作用: 定常的、τ = Cq·ω²（ファームウェアでは κ = Cq/Ct = 9.71e-3 m）
  - 加速の反トルク: 過渡的、τ = J_mp·dω/dt（J_mp = 2.01e-8 kg·m²、motor_model.cpp）
  - StampFly の数値で両者の大きさ比較の計算をやる（学生役の宿題に）
- 机の上のモータの「身震い」は角運動量保存のデモとして動画/図解向き。
- 対角 vs 隣接の説明はインタラクティブ教材の本命:
  スティック操作（ロール/ピッチ/ヨー/スロットル）→ 4モータの増減 → 合成トルクを
  矢印表示し、「隣接配置モード」に切り替えるとロール操作でヨーが漏れることを見せる。

### Q2-5. ヨー軸は他の軸より効きが弱い？

**先生の回答:**（2026-07-14）
ロールとピッチの推力によるトルクは、**推力をアームの長さで効かせられるのではるかに大きい**。
（→ ヨーは反トルク経由なので相対的に弱い。workshop の実効ゲイン Roll 102 / Pitch 70 /
Yaw 19 という5倍差と整合）

### Q2-6. ミキシングの具体形は？

**先生の回答:**（2026-07-14、導入）
クアッドコプタのミキシングはヘキサやオクタに比べれば簡単だが、**基本中の基本なので
順序立てて考える必要がある**。まず、プロペラの配置によって決まる
**アロケーション行列（配置行列）**を理解する必要がある。

**学生の試案（未確認・先生の検証待ち）:**
StampFly の幾何（motor_x,y = ±0.023 m、M1:FR CCW / M2:RR CW / M3:RL CCW / M4:FL CW、
NED座標系）から、推力 T₁..T₄ → (総推力 F, ロール τx, ピッチ τy, ヨー τz) の行列を構成:

```
[ F  ]   [  1    1    1    1  ] [T1]
[ τx ] = [ −d   −d   +d   +d  ] [T2]      d = 0.023 m
[ τy ]   [ +d   −d   −d   +d  ] [T3]      κ = Cq/Ct = 9.71e-3 m
[ τz ]   [ +κ   −κ   +κ   −κ  ] [T4]
```

- ミキシング = この行列の逆変換（クアッドは正方行列なので逆行列そのもの）では？
- 行同士が直交している（roll行·yaw行 = 0 等）ことが「対角配置ならロール操作で
  ヨーが漏れない」の数学的表現になっている。隣接配置（ヨー行 [+κ +κ −κ −κ]）だと
  roll行との内積 ≠ 0 になり干渉が出る、と教材で見せられそう。

**先生の検証:**（2026-07-14）
1. 行列は**あっている**。座標系は NED（**FRD とも言える**: Front-Right-Down）。
2. ミキシング=逆行列、ヘキサは4×6で逆が一意でない、という理解で**はい**。
3. 推力→Duty 変換も第2章で**書いておいてよい**。

**学生の追加発見（数学的に導出・章に採用）:**
隣接配置ではヨー行がロール行の κ/d 倍になり、アロケーション行列が**特異**
（行列式ゼロ = 逆行列が存在しない）。つまり干渉どころか、
**ヨーだけを独立に指令することが原理的に不可能**になる。
ロール指令はヨーを漏らし（τz_leak = (κ/d)·τx）、対角配置用のヨー指令パターン
[+,−,+,−] は隣接配置ではトルクを一切生まない（ヨー操作力の完全喪失）。

### Q2-7. 二つの反トルクの役割分担（立ち上がり=加速、定常=空気）は正しいか？

**先生の回答:**(2026-07-14)

+ヨーの空力を出すのは「機体を回したい向きと逆回転」のプロペラ（ヘリの原理で固定、
座標規約に依存しない）。そのプロペラを加速すると加速反動項も+ヨー向き

**では同定でRHP零点が「見えた」のはなぜか（見かけのRHP零点の生成機構）:**
1. **むだ時間の別名**: e^(−Ls) の1次Padé近似は (1−Ls/2)/(1+Ls/2) で

2. **モデルクラス不一致による縮退**: 進み零点+むだ時間の実プラントを

3. 非同位置の構造弾性は原理的にRHP零点を作りうるが、周波数帯が桁違いで無関係。

### Q4-16. V–I–ω 測定の具体的方法（先生の質問への提案）

**設計思想: パラメータを1つずつ孤立させる（コーストダウンで J/C_Q を電気系から
切り離したのと同じ発想）**

**試験A: Ke の単独測定（コーストダウン+電圧1ch追加、最優先・最強）**
- コーストダウン中は電流がほぼゼロ（駆動FET開、C=100nF の充電電流は µA 未満）
  → モータ端子電圧 = 逆起電力そのもの。**I·R 項が消えるので R と完全に分離**。
- 手順: いつものコーストダウン収録に CH2=ドレイン電圧を追加するだけ（2ch収録）。
  電源カット後、V_drain(t) = V_BAT − Ke·ω(t)。CH1のフォトインタラプタから ω(t)。
- 解析: V_drain vs ω の散布図が直線 → **傾き = −Ke**。減速中の数百点が1発で取れ、
  直線性そのものが「Ke=定数」の検証にもなる。V_BAT は停止後の DC レベルから読める。
- 判定: Ke ≈ 5.35e-4（論文系）か ≈ 6.125e-4（firmware系）か。差14%は余裕で分解可能。
- 注意: 停止直前の低速域はパルス間隔が伸びるので ω の補間に注意。オシロのレンジは
  V_BAT 近傍（サグやスイッチングスパイクは駆動中のみ）。

**試験B: R の測定（DC駆動・プロペラ付き・電流はシャントで）**
- PWM を使わず**DC電圧を直接**モータへ（ベンチ電源、0.5〜1.5V の3〜5点、プロペラ付き）。
  プロペラ負荷で電流が数百mAに増え、I·R 項のSNRが上がる。
- 各点で V（端子直近で測る・リード線降下を排除）、I（論文で使った R_sens=0.1Ω+アンプ、
  または DMM 直列）、ω（フォトインタラプタ）を記録。
- 解析: **R = (V − Ke·ω)/I**（Ke は試験Aの値）。複数点で一定なら確定。
  判定: R ≈ 0.593（論文系）か ≈ 0.34（firmware系）か。
- ボーナス（同じデータで Kt の独立検証）: トルク釣り合い Kt·I = C_Q·ω² + τ_c より
  **Kt = (4.10e-11·ω² + 9.5e-6)/I**。SI では Kt=Ke のはずなので、試験Aの Ke と
  一致すれば全体が閉じる（不一致なら C_Q か τ_c を疑うシグナル＝反証にも使える）。

**試験C（任意・検算）: LCR/DMM の4端子でDC抵抗**（ブラシ接触の状態依存に注意、
  回転させながらの実効値は試験Bが正）

**期待される成果:** Ke・R・Kt の三つ巴が1〜2時間の測定で確定し、τ_eff の
9.5 vs 17.5 ms 問題が解消。firmware の Dm・Qf も試験A+コーストダウン既存データで
置き換え可能（摩擦は既に τ_c=9.5e-6, B≈0 と測定済み）。
**解析スクリプトは学生役が事前に用意可能**（2ch wfm → Ke 直線フィット、
DC表 → R/Kt 表）。
