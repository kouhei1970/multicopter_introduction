### Q1. マルチコプタはなぜ「手放し」では飛べないのか？

（学生の疑問: 紙飛行機やグライダーは制御なしでも飛ぶ。気球も浮くだけなら制御は要らない。
マルチコプタは何が違って、コンピュータによる制御が「絶対に」必要なのか？）

**先生の回答:**（2026-07-13）

- 全てのプロペラが全く同じもので、同じモータが回しているなら、ひっくり返らずに飛べるかもしれない。
  しかし実際はすべて微妙に違っていて、どうしてもわずかな**推力差**が発生する。
  それによって、わずかな**姿勢を崩すトルク差**が発生する。
- わずかな違いでも、空中に浮かぶマルチコプタにとっては致命的。
  姿勢はどんどん崩れ、いずれ**ひっくり返ってしまう**。
- したがって、姿勢の崩れを**検知**して、それを**打ち消すようにプロペラの推力差を補正**してやる必要がある。
- 一方、通常の飛行機は、姿勢を崩すトルクが生じて姿勢が崩れ始めても、
  それを復元させるトルクを**機体の構造が**発生させることができる。
  - そういう復元させるトルクを**復元力**と言う。
  - 復元力が発生することを**静安定**と呼ぶ。
- 飛行機や気球には静安定があるが、マルチコプタは自らの姿勢の崩れに対して
  自らを戻す復元力を**構造上発生させることができない**。
  だから姿勢を検知してコンピュータが制御してやる必要がある。

**記事化メモ:**
- キーワード: 推力差 → トルク差 → 姿勢の崩れの発散 / 復元力 / 静安定
- 対比構造が使える: 飛行機・気球（静安定あり）vs マルチコプタ（静安定なし）
- 教材アイデア: わずかな推力アンバランスを与えた機体の 2D シミュレーション。
  「制御 OFF」だと姿勢が発散してひっくり返り、「制御 ON」にすると水平を保つトグル。
  推力アンバランス量をスライダーで変えられると効果的。

### Q1-2. 飛行機はなぜ構造だけで復元力を発生できるのか？

**先生の回答:**（2026-07-14）

- 固定翼機は**上反角**（両方の翼を上に傾ける）によって復元力を得ている。
- 仕組み: 機体がどちらかに傾くと、揚力に横方向成分が生じて機体が**横に滑り始める**。
  これによって横から風を受けることになる。
- 左右に上反角がついていると、その横から受ける風の向きに左右の翼で違いが生じる。
  翼の揚力は風の速さと向きで決定的に変わるので、横風によって**左右の翼の揚力に差**が生じ、
  それが傾きを修正する復元力となって、傾きを自動的に修正する。
- マルチコプタには揚力を稼ぐ**固定翼がない**ので、上反角効果を利用した復元力は付けられない。
- 固定翼を持つハイブリッド機も存在するが、今回の議論では除外する。

**記事化メモ:**
- 復元のメカニズムは「傾く → 横滑り → 横風 → 左右の揚力差 → 復元トルク」という因果連鎖。
  記事では矢印図か、ステップ送りのアニメーション図解が向く。
- 「静安定は翼がタダで実行してくれるフィードバック制御」という見方もできそう（先生に確認）。

### Q1-3. 姿勢の崩れはどのくらいの速さで進むのか？（StampFly で定量化）

**先生の指示:**（2026-07-14）
簡易計算に必要なのはロール軸慣性モーメント・プロペラと重心の距離・推力係数。
プロペラは角速度の2乗×推力係数に比例した推力を出す（T = Ct·ω²）。
StampFly の値を使い、左右のプロペラ角速度がわずか 0.1% ズレていた場合に
水平から 90° に傾くまでの時間を計算せよ（計算実務は学生役=AI が担当）。

**使用パラメータ（すべて先生のリポジトリの一次情報から回収）:**

| 量 | 値 | 出所 |
|----|-----|------|
| 質量 m | 36.8 g | `M5StamFly_spec_ja.md` 製品重量（firmware config.hpp は 0.037 kg でほぼ一致） |
| ロール慣性 Ixx | 9.16e-6 kg·m² | `firmware/vehicle/tasks/api_task.cpp` kSpecInertia[0]（workshop lesson_06 の表と一致） |
| モーメントアーム d | 0.023 m | `sf_algo_control/control_allocation.hpp` QuadConfig（X配置、モータ位置 ±23 mm） |
| 推力係数 Ct | 1.0e-8 N/(rad/s)² | `sf_algo_control/motor_model.cpp` |

**計算**（スクリプト: `notes/calc/roll_divergence.py`、仮定も同ファイル冒頭に明記）:

- ホバリング: 1モータあたり T₀ = mg/4 = 90.3 mN、ω₀ = √(T₀/Ct) ≈ 3004 rad/s（約 28,700 rpm）
- 左2発を ω₀(1+0.001)、右2発を ω₀ とすると、1モータあたりの推力差 ΔT ≈ 2·δ·T₀ = 0.18 mN（**18 mgf、米粒半分ほどの重さ相当**）
- ロールトルク τ = 2·ΔT·d = 8.3e-6 N·m → 角加速度 α = τ/Ixx ≈ 0.91 rad/s²
- 一定トルクなので φ(t) = ½αt²

| 傾き | 到達時間 |
|------|---------|
| 1° | 0.20 s |
| 5° | 0.44 s |
| 10° | 0.62 s |
| 45° | 1.32 s |
| **90°（ひっくり返る）** | **1.86 s** |

**考察メモ:**
- 0.1% のズレ（3004 rad/s 中のわずか 3 rad/s ≈ 29 rpm）ですら約 1.9 秒で横転。
  実機のプロペラ・モータ個体差は % オーダーのはずで（先生に要確認）、δ=1% なら時間は 1/√10 で
  **90° まで 0.59 s、10° まで 0.20 s** と人間の反応時間そのものになる。
- 人間の視覚反応（~0.2 s）で 1° の傾きに気づけたとしても、必要な補正は「スロットル差 0.1% 未満の
  精度で、連続的に、永久に」であり、プロポのスティックでは実現不可能。不安定平衡の維持は
  鉛筆を指先で立てるのと同じ構造。
- **検算として重要:** 最終結果は Ct に依存しない。ホバリング条件 Ct·ω₀² = mg/4 により
  τ = mg·δ·d となって Ct と ω₀ が打ち消し合う（Ct は ω₀ の表示にのみ使用）。
  つまり Ct の精度が結果を左右しない、頑健な見積もり。
- 未確認事項: 空力減衰・並進運動との連成・ジャイロ効果は無視した（発散初期では小さい）。
  実機の個体差 δ の実測値は未確認。

**教材アイデア（第1章の目玉候補）:**
δ をスライダーで変えると横転までの時間が変わる 2D シミュレーション。
「制御 OFF」でこの計算の通り発散し、「制御 ON」で水平を保つ対比を体感させる。
