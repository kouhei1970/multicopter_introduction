# マルチコプタ制御入門（インタラクティブ版）

GitHub Pages で公開する、触って学べるマルチコプタ制御の入門教材。

## コンセプト

- **対話から生まれる教材**: 学生役（Claude）が先生（伊藤先生）に質問し、その回答を元に記事を起草する。
  素朴な疑問 → 先生の回答 → 記事化 → インタラクティブ教材の実装、というサイクルで章を育てる。
- **副題**: 「StampFly の実例で学ぶ」（2026-07-14 先生決定）。計算・実測データ・実装例は
  一貫して StampFly を題材にする。
- **対象読者**: 制御工学を学ぶ大学生以上。伝達関数・状態方程式は前提としてよい。
- **ゴール**: モデリングから PID・状態推定・カスケード制御を経て、実機を飛ばすファームウェアの世界まで。
- **形式**: 章立ての複数ページ。各章に JavaScript によるインタラクティブ教材（スライダーでゲインを変えると応答が変わる図、推力操作シミュレーション等）を埋め込む。

## 技術方針

- ビルド不要の素の HTML + CSS + JavaScript（GitHub Pages にそのまま置くだけで動く）
- 数式は KaTeX（ブラウザ上で LaTeX を組版するライブラリ）を CDN から読み込み
- シミュレーションは Canvas / SVG + requestAnimationFrame で自前実装（外部依存を増やさない）

## 章構成（ドラフト・対話しながら更新する）

|----|---------|------|
| 1 | マルチコプタはなぜ落ちないのか — 導入と全体像 | 対話中 |
| 2 | 機体のしくみ — ロータ配置・推力とトルク・ミキシング | 未着手 |
| 3 | 運動のモデリング — 座標系と剛体の運動方程式 | 未着手 |
| 4 | モータとプロペラ — 推力・反トルクと応答遅れ | 未着手 |
| 5 | 姿勢の PID 制御 — レート制御からカスケードへ | 未着手 |

※ この構成は学生役の「学びたい順」の仮説。対話の中で先生の判断でいくらでも組み替える。

## ディレクトリ構成

```
index.html        # 目次・ランディングページ
chapters/         # 各章の HTML
assets/css/       # 共通スタイル
assets/js/        # インタラクティブ教材の共通 JavaScript
notes/            # 対話メモ（先生の回答の生記録。記事の原材料）
```

## ワークフロー

1. 学生役が質問 → 先生が回答（チャット上）
2. 回答を `notes/qa_log.md` に記録
3. たまった材料から章の HTML を起草し、インタラクティブ教材を実装
4. ブラウザで確認 → 修正 → コミット → GitHub Pages で公開
